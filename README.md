# CS-300
Portfolio Reflection
What was the problem you were solving in the projects for this course?
The projects in this course focused on designing a program that could efficiently store, organize, and retrieve course information for an academic advising system. The final application needed to load course data from a file, validate prerequisites, and print the Computer Science courses in alphanumeric order while allowing users to search for individual courses. The goal was to determine which data structure would provide the best balance of efficiency and functionality for the required operations.

How did you approach the problem? Consider why data structures are important to understand.
I compared the different data structures to determine which one would work best for the ABCU advising project. I considered not only the speed of inserting and searching data, but also how efficiently each data structure could keep the course information sorted. After comparing the options, I chose a Binary Search Tree (BST) because it provided a good balance of performance while naturally storing the courses in alphabetical order, making it the best fit for the project requirements.

How did you overcome any roadblocks you encountered while going through the activities or project?
The biggest roadblock I encountered was getting the course data to load correctly into the BST. I originally tried loading the CSV file from a folder on my computer but continually ran into issues. At one point, Codio crashed, which caused Visual Studio to display errors that were not actually present in my code. I spent several hours trying to debug a problem that didn't exist before realizing the development environment was causing the issue. After resetting the Codio IDE, I decided to place the CSV file in the project's resources folder, similar to how the other course projects were organized. Once I made that change, the program loaded the data correctly. Another challenge was ensuring the BST maintained the proper order so the courses printed in alphanumeric order, which required careful testing and debugging.

How has your work on this project expanded your approach to designing software and developing programs?
This project reinforced that choosing the right data structure is about more than selecting the fastest option. It is important to consider how the data will be used throughout the program. I chose a Binary Search Tree because it not only performed efficiently but also supported the project's requirement to print the courses in alphanumeric order. This experience taught me to evaluate data structures based on both performance and how well they meet the specific needs of the application.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
Throughout this project, I focused on writing modular code by separating functionality into classes and functions with clear responsibilities. I also used meaningful variable names, comments where appropriate, and consistent formatting to improve readability. Breaking the program into smaller components made it easier to debug, update, and extend.
